Choose a connection

Select the Microsoft Fabric connection this data share writes to. The store type belongs to the connection — a GraphQL API writes to exactly one database, so tables can't be split across stores.

Search existing connections
Select a connection to add

Pick an existing Microsoft Fabric connection (connections are created and edited on the Connections tab — not here).

This connection writes to Data Warehouse via the gateway. The store type is set on the connection and can't be changed here.

Choose an app

Pick the Atlassian app whose data you want to share.

Source data

Choose which Atlassian data flows to Fabric — and how much of it. This is the source side; the target store was set on step 1.

Target tables

These tables map to your selection. All are active by default — deactivate any you don't need. At least one must stay selected.

Set the refresh frequency

How often new and changed data is pushed to Fabric.
